# infra/.codocs/

codocs 工具包目录。由 `codocs init` 安装到下游项目，提供文档完整性检查和 git hook 拦截能力。

## 索引

| 名称 | 类型 | 职责 |
|---|---|---|
| `codocs.py` | 文件 | 核心引擎：扫描、lint、哈希计算、review/refresh、bootstrap |
| `hooks/` | 目录 | git hook 静态脚本（pre-commit、commit-msg）|

## 安装到下游项目

`codocs init` 将 `codocs.py` 复制到项目的 `.codocs/scripts/codocs.py`，将 hooks 注册到 `.git/hooks/`（或通过 more-hooks 注册到 `hooks.d/`）。

安装后，infra 原件与项目本地副本**相互独立**：项目使用自己 `.codocs/scripts/` 下的版本，升级需重新 init。

## 架构定位

在 infra/ 三工具中，`.codocs/` 提供**文档质量闸门**，`.more-hooks/` 提供**hook 调度框架**，`.ucmake/` 提供**构建质量闸门**。三者通过 more-hooks dispatcher 串联，互不干扰。
